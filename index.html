01 / Overview

A small cloud, owned at home.

Immich is an open-source photo and video management platform. It feels familiar if you have used Google Photos, but the important difference is ownership: the server runs on my own machine instead of someone else's cloud.

For this project, I kept the setup intentionally simple: Zorin OS as the host operating system, Docker Compose to run the services, and the Immich web interface available on the local network through port 2283.

What it doesStores photos and videos, organizes them, and allows access from a browser or mobile app.
Why it mattersThe data stays on hardware I control, with less dependence on external cloud storage.
How it runsDocker Compose starts Immich's services, including the web server, machine learning service, Redis/Valkey, and Postgres database.
How users connectDevices on the same network open http://<server-ip>:2283.
02 / Purpose

Ownership feels different.

Selfhosting is cool because it makes technology feel real. Instead of depending on a service I barely control, I can run the software myself, understand the moving parts, and decide how my own data is stored.

Privacy does not mean having something to hide. For me, privacy is a principle. It is about respecting personal space, protecting family memories, and choosing not to give away more information than necessary just because the default option makes it convenient.

The point of this project was not to overcomplicate the home lab. It was to build something simple, useful, and repeatable: a server that receives uploads from phones and makes those photos available through a clean private interface.

Simple idea: keep useful services close, understandable, and under your control.
03 / Requirements

Small requirements. Real utility.

Immich can run on many systems, but it should not be treated like a tiny background script. It processes photos, generates thumbnails, indexes metadata, and runs a database. The official quick-start path recommends at least:

RequirementRecommended minimumReason
CPU2 cores or betterPhoto processing and background jobs need compute power.
Memory6GB RAM or betterImmich runs multiple containers and services.
StorageEnough space for your libraryPhotos and videos can grow quickly over time.
SoftwareZorin OS + Docker Engine + Docker Compose pluginImmich requires Docker with the modern Compose plugin for the recommended deployment path.
Practical note: for a real family photo server, storage planning matters more than the install itself. The installation is simple; the long-term value depends on having enough disk space and a backup plan.
04 / Docker Setup

Prepare Docker before Immich.

Immich's recommended installation path uses Docker Compose, so the server needs Docker Engine and the Docker Compose plugin before the Immich files are started.

Step 1 — Download and run Docker's official convenience script

curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

Docker's convenience script is a quick way to install Docker on Linux by detecting the distribution, configuring Docker's package source, installing required packages, and preparing the Docker service.

Fact check: this is Docker's convenience script, not Immich's install script. Docker documents it for quick non-interactive setup, while Immich recommends Docker Compose for production deployments.

Step 2 — Verify Docker and Compose

After installation, Docker was checked from the terminal to confirm that the command, service, and Compose plugin were available. 

sudo docker --version
sudo docker compose version
sudo systemctl status docker
Verification: docker --version confirms the Docker CLI, docker compose version confirms the Compose plugin, and systemctl status docker confirms that the Docker service is running.

Why Docker Compose?

Immich is made of multiple services, including the web application, database, machine learning service, and cache layer. Docker Compose keeps those services organized and allows the full Immich stack to run from a single configuration file.

Docker Engine→ Compose Plugin→ PostgreSQL / Redis→ Immich Services
05 / Network

Local first, easy to reach.

The Immich server lives on the local network like any other device. The router gives it a local IP address, and other devices connect to that IP using Immich's web port: 2283.

Access pattern

http://<server-ip-address>:2283

Example: if the Zorin OS machine receives the local IP 192.168.1.50, the Immich web app would be opened at http://192.168.1.50:2283.

Recommended router setting

The cleanest setup is to create a DHCP reservation on the router. That means the router always gives the Zorin OS server the same local IP address. This avoids the server IP changing later and breaking the saved Immich address on phones or browsers.

Keep it simple: this page describes a local-network setup. If this server is ever exposed outside the home network, it should be done carefully with HTTPS, strong authentication, updates, and a real security plan.
06 / Immich Setup

Install Immich with Docker Compose.

Immich officially recommends Docker Compose for production. The process is simple: create a project folder, download the official compose and environment files, customize the environment values, and start the stack in the background.

Official reference: this section follows Immich's Docker Compose recommended installation path. Immich's separate install script is marked experimental and is not the recommended production method.

Step 1 — Create the project folder

mkdir ./immich-app
cd ./immich-app

Step 2 — Download the official files

wget -O docker-compose.yml https://github.com/immich-app/immich/releases/latest/download/docker-compose.yml
wget -O .env https://github.com/immich-app/immich/releases/latest/download/example.env

The second command downloads Immich's example environment file and saves it directly as .env, which is the filename Docker Compose expects.

Step 3 — Edit the .env file

The key values are the media upload location, database location, timezone, Immich version, and database password. 

# Where uploaded photos and videos are stored
UPLOAD_LOCATION=./library

# Where database files are stored
DB_DATA_LOCATION=./postgres

# Set your timezone by uncommenting and editing this value
TZ=America/New_York

# Immich version. This can be pinned to a specific release if needed.
IMMICH_VERSION=v2

# Change this to a custom alphanumeric password
DB_PASSWORD=ChangeThisPassword123
Important: UPLOAD_LOCATION should point to a new directory with enough free space. Immich also recommends changing DB_PASSWORD to a custom value using only A-Za-z0-9 characters.

Step 4 — Start the containers

docker compose up -d

Run this from the folder containing docker-compose.yml and .env. Docker Compose starts Immich as a background service, and the web app becomes available on port 2283 once the containers are healthy.

Command note: the correct modern command is docker compose, not docker-compose. If Docker reports a Compose or health check error, verify that Docker Engine and the Compose plugin are current.
07 / Web App

The first account becomes home base.

The first account created in Immich becomes the admin user. From a browser on the same network, open the server address: 

http://<server-ip-address>:2283
  1. Open the Immich web app in a browser.
  2. Click Getting Started.
  3. Create the first user account. This becomes the admin account.
  4. Log in and upload a test photo from the browser.

After this, the server is usable. You can upload from the web UI, manage users, and confirm that the storage path is working before moving on to phone backups.

08 / Mobile Backup

The phone is where it clicks.

The web app is great for testing, but the real value is mobile backup. Install the Immich app on your phone, connect it to the server URL, choose which albums to back up, and enable backup.

StepAction
1Install the Immich mobile app from the app store or official release source.
2Enter the server endpoint: http://<server-ip-address>:2283.
3Log in with the user account created on the web app.
4Open the backup screen using the cloud icon.
5Select the albums to back up and press Enable Backup.
Good first test: upload only a few photos at first. Once the server processes them correctly, enable larger album backups.
09 / Backups

Keep the memories backed up.

Immich stores and organizes the media, but the server still needs a real backup strategy. The database contains metadata and user information. The actual photos and videos live in the path configured as UPLOAD_LOCATION.

Do not skip this: back up both the database and the uploaded media files. If you only back up the database, you do not have the photos. If you only back up the photos, you may lose metadata, users, albums, and application state.
DatabaseContains metadata, users, albums, and Immich application information.
Upload libraryContains the actual photos and videos. This is the most important data.
Final result

A useful system, not just a lab experiment.

This project was worth it because it stayed understandable. The server runs locally, the services are organized with Docker, the network setup is simple, and phones can back up directly to a private photo library at home.
